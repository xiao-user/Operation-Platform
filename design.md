# 智慧校园运营管理平台 SaaS Design Guidelines

本文档描述当前 SaaS Shell 及其内部业务页面的视觉、布局与交互规范，用于新增页面、重构现有页面和评审设计一致性。

日常业务开发默认只负责 `.app-content-inner` 内的内容区域。顶部栏、模块导航轨、深层菜单、租户切换和角色切换由持续挂载的 SaaS Shell 统一提供；除非需求明确修改平台导航，否则业务页面不得复制、重排或侵入这些区域。

规范以当前代码为事实源，主要对应：

- `src/layouts/AppLayout.vue`
- `src/components/AppHeader.vue`
- `src/components/AppModuleRail.vue`
- `src/components/AppSidebar.vue`
- `src/assets/main.css`
- 已有工作台、审核列表、组织配置和学生成长画像页面

区域教育总览、智慧体育数据驾驶舱等 `standalone` 大屏不属于本文档的 SaaS Shell 范围，应遵循各自的数字孪生设计系统。

## 1. 设计目标

平台面向教育局、学校、机构和平台运营人员，页面应优先满足高频管理任务，而不是追求装饰性表达。

核心原则：

1. **层级清楚**：导航、筛选、操作和数据内容的位置稳定，用户无需重新学习每个页面。
2. **信息紧凑但不拥挤**：以 14px 正文和 16px 基础间距承载管理型信息密度。
3. **扁平克制**：主要依靠背景、细边框和间距分层，不使用大面积阴影、渐变或过度圆角。
4. **状态明确**：当前租户、角色、导航位置、筛选状态、保存状态和异常状态必须可辨认。
5. **组件优先**：优先使用 Element Plus 和仓库已有公共组件，不在页面中重复实现按钮、筛选栏、状态标签和弹窗行为。
6. **响应式可用**：窄屏可以降级，但不能遮挡核心操作、产生不可滚动区域或破坏路由状态。

## 2. 页面总体构图

### 2.1 Shell 分区

SaaS Shell 统一提供顶部栏、左侧模块导航轨、深层菜单、主内容区和可选的右侧 AI 助手。业务页面只渲染主内容区。

业务开发需要知道的边界只有：

- Shell 占满 `100vh`，主内容区承担页面滚动。
- 主内容区会随左侧导航和右侧 AI 助手动态收缩，页面不得假设固定可用宽度。
- 页面根节点必须允许收缩，满高表格等场景由页面内部管理剩余高度。
- 工作台和普通业务页仍由同一 Shell 提供导航。
- `standalone` 页面不挂载本 Shell，也不适用本文的内容区布局模板。

### 2.2 工作台构图

工作台是 Shell 的特殊状态：

- 隐藏左侧模块导航轨和深层菜单。
- 顶部栏展示一级模块入口。
- 内容区桌面端内边距为 `16px 24px`。
- 工作台组件使用独立布局系统，不得复用菜单节点模拟组件。
- 窗口宽度低于 `1200px` 时，工作台布局调整降级为只读。

### 2.3 普通业务页构图

普通页面放置在 `.app-content-inner` 中，默认内边距为 `16px`。

页面根容器必须至少满足：

```css
.page {
  min-width: 0;
  min-height: 0;
}
```

需要满高的管理页面使用：

```css
.page {
  display: flex;
  flex-direction: column;
  height: 100%;
  min-width: 0;
  min-height: 0;
}
```

内容自然增高的分析页使用 Grid，不强制占满视口：

```css
.page {
  display: grid;
  min-width: 0;
  gap: var(--spacing-16);
}
```

## 3. 布局规则

### 3.1 间距体系

所有页面使用既有间距变量，不直接编写重复的任意数值。

| 场景 | 建议间距 |
| --- | --- |
| 图标与文字 | `6px` 或 `8px` |
| 同组紧密控件 | `8px` 或 `12px` |
| 卡片内部元素 | `12px` 或 `16px` |
| 页面模块之间 | `16px` |
| 筛选项横向间距 | `24px` |
| 页面主体内边距 | `16px` 或 `24px` |
| 大区块分隔 | `24px` 或 `32px` |

规则：

- 默认以 `4px` 为基础步长。
- 同一层级的水平和垂直间距应保持一致。
- 不使用空 `div`、连续 `<br>` 或不可解释的 margin 制造布局。
- 页面局部间距优先使用 `gap`，减少相邻选择器和最后一项清零逻辑。

### 3.2 页面栅格

SaaS 页面不使用固定 12 栅格作为强制约束，采用内容驱动的 CSS Grid/Flex。

推荐模式：

```css
/* 自适应卡片区 */
grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));

/* 左侧专题导航 + 主内容 */
grid-template-columns: 168px minmax(0, 1fr);

/* 主区域 + 辅助区域 */
grid-template-columns: minmax(0, 2fr) minmax(280px, 1fr);
```

约束：

- 所有可伸缩列都使用 `minmax(0, 1fr)`，避免长文本撑破页面。
- Flex 子项需要收缩时必须设置 `min-width: 0`。
- 满高 Flex/Grid 子项需要内部滚动时必须设置 `min-height: 0`。
- 同一行卡片可以等高；跨行内容不强制等高。
- 数据卡片默认由内容撑高，只有明确的图表、表格或滚动区域才设置最小高度。
- 内容页面不得产生页面级横向滚动；宽表、图表等溢出必须收敛在自身容器内。
- 不通过绝对定位搭建普通业务页面主体。

### 3.3 标准列表页

列表型管理页按照以下顺序构成：

```text
筛选栏
└─ 筛选字段 + 查询操作

主体白底区域
├─ 工具栏：页面/列表标题 + 主次操作
├─ 表格：占据剩余高度
└─ 分页（按业务需要）：右对齐
```

具体规则：

- 筛选栏复用 `PageFilterBar`，白色背景、底部分割线、内边距 `16px 24px`。
- 筛选字段允许自动换行，横向间距 `24px`，纵向间距 `16px`。
- 常规“标签 + 控件”筛选项宽度为 `280px`，控件高度为 `32px`。
- 主体区使用白色背景和 `24px` 内边距。
- 工具栏与表格间距为 `16px`。
- 表格容器使用 `flex: 1; overflow: hidden`，表格本身根据需求承担纵向或横向滚动。
- 需要分页时，分页位于主体右下方，不与表格浮层重叠。
- 页面级主要操作放在工具栏右侧，最重要操作使用 Primary Button。

### 3.4 分析与画像页

分析页面采用“筛选 + 视图导航 + 内容面板”：

- 分析与画像页属于 SaaS 内容区的常规数据分析页面，与独立数据大屏无关；不得复用 HUD、驾驶舱构图、沉浸式主题或大屏交互范式。
- 全局筛选放在内容最上方，不重复出现在每张卡片中。
- 筛选项只保留能改变统计范围且具备明确数据字段的维度；不得用无法稳定计算的“数据状态”或与学校范围重复的行政维度制造虚假能力。
- 业务术语使用领域通行名称，例如小学、初中、高中统一称为“学段”；年级选项应随学段联动并覆盖该学段的完整年级范围。
- 同一分析主题的二级视图使用 `ElSegmented`、Tabs 或左侧专题菜单。
- 左侧专题菜单桌面端建议宽度 `168px`，窄屏可收缩至 `136px`。
- 长内容页面的左侧专题菜单使用 `position: sticky; top: 0`；最小高度按“视口高度减 Shell 顶栏和内容区上下边距”计算，最大高度使用同一结果，超出时仅菜单内部纵向滚动。
- 内容区保持 `16px` 模块间距。
- 数据说明、口径和敏感信息规则使用 Drawer 承载，避免持续占用主内容空间。
- 图表解读、统计口径等说明性文字不单独占用内容模块；放在对应标题右侧的说明图标中，通过 Tooltip 展示。
- 当前专题已经由侧边菜单明确表达时，不在内容区重复渲染同名页面标题。
- 页面中使用模拟数据时必须明确标识，不得让模拟数据看起来像实时生产数据。
- 跨主题综合指数只有在权重、公式、分母和适用范围明确时才可进入核心摘要；否则优先展示覆盖学校、覆盖学生和待跟进事项等可追溯事实。
- 不展示无业务处理链路的操作按钮，也不把纯流程说明伪装成可点击控件。

### 3.5 表单与详情页

- 简单表单优先使用单列；字段较多时使用两列 Grid。
- 单个字段标签和控件应建立稳定对齐关系，不依赖空格字符。
- 详情信息优先使用 `ElDescriptions`，编辑行为使用表单或 Drawer/Dialog。
- 保存、取消放在表单末尾或固定操作区；主要操作在右侧。
- 危险操作不得与主要保存按钮同色、同位置。
- 表单错误显示在字段附近，页面级失败通过 `ElAlert` 或 `ElMessage` 补充。

### 3.6 卡片与面板

- SaaS 内容面板默认白底。
- 普通功能面板圆角使用 `4px`；独立卡片、浮层和工作台组件可使用 `8px`。
- 相邻、连续的数据区域优先使用分割线，不应把每个小指标都包装成悬浮卡片。
- 阴影仅用于浮层、弹出层或需要明确悬浮关系的组件。
- 面板标题使用 `16px / 24px / 600`，操作位于同一行右侧。
- 卡片内容过多时优先增加内部结构或展开入口，不盲目增加固定高度。

## 4. 视觉规范

### 4.1 色彩

主品牌色：

| Token | 值 | 用途 |
| --- | --- | --- |
| `--color-primary` | `#2D55EB` | 主要按钮、选中态、链接 |
| `--color-primary-hover` | `#4A6EF0` | Hover |
| `--color-primary-light` | `#EAEEFE` | 浅色选中背景 |
| `--color-primary-dark-text` | `#1842D3` | 深色强调文字 |

中性色：

| Token | 值 | 用途 |
| --- | --- | --- |
| `--color-title` | `#252526` | 标题、主要文本 |
| `--color-body` | `#575859` | 正文 |
| `--color-secondary` | `#898A8C` | 辅助信息 |
| `--color-placeholder` | `#BBBCBF` | 占位和禁用文字 |
| `--color-bg` | `#F6F7F9` | Shell 页面背景 |
| `--color-white` | `#FFFFFF` | 导航和内容面板 |
| `--color-border` | `#EBECF0` | 常规分割线 |

状态色只表达语义：

- 成功：绿色。
- 警告：橙色。
- 错误或危险：红色。
- 信息与默认选中：蓝色。
- 中性或不可用：灰色。

禁止仅使用颜色表达状态，应同时提供文字、图标、形状或可访问名称。

### 4.2 字体

字体栈：

```css
"PingFang SC", "Microsoft YaHei", -apple-system, BlinkMacSystemFont, sans-serif
```

| 层级 | 字号 / 行高 | 字重 | 用途 |
| --- | --- | --- | --- |
| 页面或模块标题 | `16px / 24px` | `600` | 工具栏标题、面板标题 |
| 正文与表格 | `14px / 22px` | `400` | 默认内容 |
| 强调正文 | `14px / 22px` | `600` | 当前租户、用户名、选中项 |
| 辅助文本 | `13px / 20–22px` | `400` | 说明、次级信息 |
| 标签与注释 | `12px / 20px` | `400–500` | 状态、数据口径 |

规则：

- 不使用超大标题制造营销页效果。
- 中英文、数字混排保持同一基线。
- 单行导航和表格内容默认省略并提供 Tooltip 或详情入口。
- 数据值可提高字重，但不要随意增加字号层级。

### 4.3 圆角、边框与阴影

| Token | 值 | 使用场景 |
| --- | --- | --- |
| `--radius-sm` | `2px` | 小标签、焦点轮廓辅助 |
| `--radius-md` | `4px` | 输入、按钮、普通面板 |
| `--radius-lg` | `8px` | 工作台卡片、浮层、独立卡片 |
| `--radius-full` | `999px` | 头像、圆形状态 |

- 页面结构分区使用 `1px solid var(--color-border)`。
- 不为普通内容面板同时叠加粗边框和阴影。
- `--shadow-s/m/l` 仅用于 Dropdown、Popover、Dialog 等浮层关系。

## 5. Shell 与导航边界

- 正常业务开发不修改 Header、模块导航轨和深层菜单的布局或样式。
- 页面只通过 `page-registry.ts` 注册资源，通过菜单的 `pageKey` 获得入口。
- 页面内跳转使用 Vue Router；外部链接才使用普通 `<a>`。
- 页面不得自行实现租户、角色、权限或当前导航高亮逻辑。
- 只有明确的 Shell 改版需求，才进入导航组件范围，并需要单独验证全租户、全角色和响应式行为。

## 6. 组件使用规则

### 6.1 Element Plus

优先使用：

- `ElButton`、`ElInput`、`ElSelect`、`ElCheckbox`、`ElSwitch`
- `ElTable`、`ElPagination`
- `ElForm`、`ElDescriptions`
- `ElDialog`、`ElDrawer`、`ElDropdown`
- `ElTabs`、`ElSegmented`、`ElMenu`
- `ElAlert`、`ElMessage`、`ElMessageBox`
- `ElEmpty`、`ElSkeleton`

不得仅为了微小视觉差异重新实现成熟组件。需要覆盖样式时先通过全局 Token 和 Element Plus CSS 变量处理。

### 6.2 按钮

- 一个操作组中最多只有一个主要按钮。
- 次要操作使用默认按钮；文字型低风险操作可用 Link Button。
- 删除、停用等使用 Danger 语义并提供确认。
- 图标用于增强识别，不替代按钮文字；纯图标按钮必须设置 `aria-label` 和 `title`。
- 按钮顺序遵循“次要在左、主要在右”。

### 6.3 筛选与输入

- 输入控件默认高度遵循 Element Plus，紧凑筛选区使用 `32px`。
- Placeholder 描述输入内容，不代替字段标签。
- 搜索条件较多时允许换行，不横向压缩到不可读。
- “查询”使用主要按钮；“重置”使用次要按钮。
- 已确定的查询条件直接放入筛选栏并自动换行；操作区只保留“查询”和“重置”，不再增加“更多筛选”入口。
- 移除折叠筛选入口时，原有条件必须迁入筛选栏或连同状态完整删除，不得留下用户无法访问的隐藏状态。
- “数据说明”属于数据治理类页面的特殊辅助入口，可以与查询、重置并列保留。
- 清空筛选后必须回到明确的默认范围。

### 6.4 分段控件

- 分段切换优先直接使用 Element Plus `ElSegmented`。
- 不修改其 DOM、内边距、滑块位置、动效或尺寸结构。
- 圆角继承全局 `--el-border-radius-base`，颜色通过 Element Plus 的 `--el-segmented-*` 变量映射到设计 Token。
- 分段控件只用于同一数据范围内的少量互斥视图，不代替页面导航和多级菜单。

### 6.5 表格

- 表头使用浅灰背景和半粗字重。
- 行分割线使用 `--color-border`。
- 可使用斑马纹，但不要同时叠加强 Hover、卡片边框和多色背景。
- 状态列使用文字加状态点或 `StatusTag`。
- 数字列按业务含义右对齐；名称和描述左对齐；序号可居中。
- 空数据使用 `ElEmpty`，加载使用 `v-loading` 或 Skeleton。

#### 表格基础属性契约

表格及公共表格封装必须显式传递稳定属性，不依赖列的显示顺序或表头文字识别业务含义。

```ts
interface BaseTableColumn {
  key: string;
  label: string;
  prop?: string;
  columnKey?: string;
  type?: "default" | "selection" | "index" | "expand";
  width?: number;
  minWidth?: number;
  fixed?: "left" | "right";
  align?: "left" | "center" | "right";
  showOverflowTooltip?: boolean;
}
```

规则：

- 每个普通数据列必须有稳定的 `key`，并传入 `prop` 或 `column-key`；使用 Slot 自定义内容时也不能省略稳定标识。
- 列标题统一通过 `label` 传入，禁止在公共表格内部根据字段名猜测中文标题。
- 固定列必须显式传入 `fixed="left"` 或 `fixed="right"`，不得通过 CSS `position: sticky` 另做一套固定逻辑。
- 固定宽度列传 `width`；内容可能变化的列传 `min-width`，不得同时省略导致表格随机分配关键列宽。
- 文本可能溢出的数据列默认传 `show-overflow-tooltip`。
- 排序、筛选和服务端查询字段使用稳定的 `prop` 或 `column-key`，不得使用中文 `label` 作为接口参数。
- 表格必须传入稳定 `row-key`；不得使用当前页数组下标作为数据身份。

#### 基础列顺序

标准顺序为：

```text
多选列（可选） → 展开列（可选） → 序号列 → 主标识列 → 普通数据列 → 状态列 → 操作列（可选）
```

并遵循：

- 多选列仅在存在批量操作时展示，使用 `type="selection"`、`width="48"`、`fixed="left"`。
- 跨分页保留选择时，表格必须设置稳定 `row-key`，多选列必须传 `reserve-selection`；页面保存的是业务主键集合，而不是行对象引用。
- `selectable` 等禁选规则由业务层显式传入，并为禁选原因提供 Tooltip 或辅助说明。
- 序号列建议 `width="64"`、`align="center"`；分页序号根据页码和每页数量计算。
- 名称、编号等主标识列在横向滚动时需要持续可见，通常设置 `fixed="left"` 和合理的 `min-width`。
- 操作列完全由业务需求决定；不得为了表格形式统一而添加“查看”“详情”或空操作列。
- 业务确实存在独立行级操作时，操作列位于最后并设置 `fixed="right"`，操作文本使用品牌色。
- 已存在整行点击、主标识链接等明确入口时，应评估是否还需要重复的操作列。
- 使用操作列时，一个操作建议宽度 `96px`，两个操作建议 `160px`；三个及以上应将低频操作收入“更多”，避免无限增加固定列宽。

推荐结构：

```vue
<el-table
  v-loading="loading"
  :data="rows"
  row-key="id"
  stripe
  border
  height="100%"
>
  <el-table-column
    v-if="batchActions.length"
    type="selection"
    width="48"
    fixed="left"
    reserve-selection
    :selectable="isRowSelectable"
  />
  <el-table-column type="index" label="序号" width="64" align="center" fixed="left" />
  <el-table-column
    prop="name"
    column-key="name"
    label="名称"
    min-width="180"
    fixed="left"
    show-overflow-tooltip
  />
  <el-table-column prop="status" column-key="status" label="状态" width="120" />
  <!-- 仅在业务需要独立行级操作时添加 -->
  <el-table-column v-if="hasRowActions" label="操作" width="160" fixed="right">
    <template #default="{ row }">
      <!-- 行级操作 -->
    </template>
  </el-table-column>
</el-table>
```

#### 分页使用条件

分页不是表格的必选组成，是否展示由数据规模、读取方式和业务任务决定。

- 远端接口已经分页、总量可能持续增长，或完整数据无法在当前页面高效浏览时使用分页。
- 固定且数量较少的指标详情、配置摘要、步骤记录和说明性数据可以完整展示，不添加分页。
- 不得为了页面形式统一给五条、十条固定数据机械添加分页。
- 使用分页时默认与审核列表保持一致：右对齐、`background`，布局为 `total, sizes, prev, pager, next, jumper`。
- 默认每页数量为 `10`，可选项为 `[10, 20, 50, 100]`；业务确有不同信息密度或接口限制时可以调整，但同一页面内必须一致。
- 页容量变化后应回到第一页或由 Store/Repository 校正合法页码，避免出现空白页。
- 标准分页已经展示总数时，不在左侧重复编写另一套“共多少条”的自定义说明；额外业务总量应放在表格标题或摘要区。

### 6.6 状态与反馈

- 页面加载：局部加载优先，不冻结整个 Shell。
- 保存成功：简短 `ElMessage.success`。
- 可恢复错误：在对应模块内显示 `ElAlert` 并提供重试。
- 删除或放弃修改：使用 `ElMessageBox` 二次确认。
- 空状态必须说明原因，并在可操作时给出下一步按钮。
- 使用模拟数据、过期数据或只读状态时，必须持续可见地标识。

## 7. 交互与动效

- Hover、选中和展开过渡通常为 `160–200ms`。
- Shell 只对模块导航轨和 AI 助手等结构变化使用轻量动效。
- 不增加无业务意义的路由切换动画。
- 动效优先使用 `opacity` 和 `transform`，避免高频触发布局计算。
- 必须支持 `prefers-reduced-motion: reduce`，关闭非必要过渡。
- 抽屉、弹窗关闭后应恢复触发元素焦点。
- 所有可点击非原生元素应改为 `button`、`a` 或 `RouterLink`。

## 8. 响应式规则

### 小于等于 767px

- 顶部品牌区缩为 `56px`，隐藏机构名称。
- 顶部仅保留当前活动页签，字号降为 `14px`。
- 隐藏角色和用户文本操作，保留必要租户信息。
- 工作台内容内边距降为 `12px`。
- AI 助手覆盖顶部栏以下的完整视口宽度。
- 多列内容改为单列；表格保留横向滚动，不压缩关键列。

### 768px–1100px

- AI 助手宽度为 `min(419px, 42vw)`。
- 主辅双列页面应评估切换为单列或缩窄辅助栏。
- 筛选项继续自动换行。

### 小于 1200px

- 工作台布局编辑不可用，展示自动适配后的只读布局。

响应式实现不得：

- 通过整体缩放页面适配。
- 隐藏主要业务操作而不给替代入口。
- 让多个区域同时产生不可控的双重纵向滚动。

## 9. 可访问性

- 所有图标按钮提供可访问名称。
- 当前导航使用 `aria-current="page"`。
- 展开控件同步表达展开状态。
- 焦点态使用 `2px` 浅品牌色轮廓，不直接移除键盘焦点。
- 表单字段具有可关联标签。
- 错误提示不能只依靠颜色。
- 文本和背景保持可读对比度。
- 键盘应能完成导航、筛选、菜单展开、弹窗确认和表格主要操作。

## 10. 新页面落地清单

新增或评审 SaaS 页面时逐项检查：

- [ ] 页面是否继续挂载 `AppLayout`，而不是复制 Header/Sidebar。
- [ ] 页面是否注册在 `page-registry.ts`，导航是否只保存 `pageKey`。
- [ ] 是否采用列表页、分析页、表单页或工作台中的既有布局模式。
- [ ] 是否使用 `min-width: 0`、`min-height: 0` 正确处理伸缩和内部滚动。
- [ ] 是否只使用现有颜色、间距、字号、圆角和阴影 Token。
- [ ] 是否优先复用 Element Plus 与公共组件。
- [ ] 表格是否只按业务需要展示多选列和操作列；存在这些列时是否传入明确的类型、宽度、对齐与 `fixed` 属性。
- [ ] 跨分页多选是否同时具备稳定 `row-key`、`reserve-selection` 和业务主键集合。
- [ ] 是否只有一个视觉主要操作。
- [ ] 加载、空数据、错误、禁用、无权限和未保存状态是否完整。
- [ ] 交互是否支持键盘、焦点和 `prefers-reduced-motion`。
- [ ] 767px、1100px 和 1200px 附近是否经过响应式验证。
- [ ] 是否避免影响 `.regional-digital-twin` 等独立大屏样式域。

## 11. 禁止事项

- 不在业务页面重新创建一套品牌色、间距或字体变量。
- 不复制 Shell、导航、租户切换或角色切换逻辑。
- 不因单个业务页面需求修改顶部栏、模块导航轨或深层菜单结构。
- 不用大量大圆角、渐变、玻璃态和阴影改变当前 SaaS 风格。
- 不在视图中直接写权限、租户、菜单或持久化领域规则。
- 不用固定像素宽度锁死主内容区。
- 不为每个页面创建不同的筛选栏、表格操作和状态标签风格。
- 不让 SaaS 全局样式影响独立数字孪生页面，反之亦然。
